# Example — General Conversation Map (Light)

This is what a Context Map looks like for a discussion or advice session.

## Scenario
User discussing a career transition from Software Engineering to Product Management.
Session ended after covering pros/cons and salary impact — next step was portfolio building.

---

## The Map

```
╔══════════════════════════════════════════════════════╗
║  CONTEXT MAP V2.1                                    ║
║  Type: Conversation  |  Size: Light                  ║
║  Session: SE to PM Career Switch                     ║
╠══════════════════════════════════════════════════════╣
║  LAYER 1 — GOAL                                      ║
║  Transition from Software Engineering to Product     ║
║  Management internally within 6 months               ║
╠══════════════════════════════════════════════════════╣
║  LAYER 3 — STATE                                     ║
║  ✅ DONE    : Pros/cons analysis complete             ║
║               Salary impact discussed (~10-15% drop) ║
║               Internal vs external transfer compared ║
║  🔄 PENDING : How to build PM portfolio while in SE  ║
║  ⚠️ BLOCKER : Manager support for transition unknown ║
║  💡 LOCKED  : Internal transfer first (not external) ║
║               6 month transition timeline            ║
╠══════════════════════════════════════════════════════╣
║  LAYER 4 — LESSONS                                   ║
║  🔒 CRITICAL: APM programs not suitable — too junior ║
║               for user's experience level            ║
║  🔁 PATTERNS: Concrete action steps only — no theory ║
║               User wants specific, practical advice  ║
╚══════════════════════════════════════════════════════╝
📍 Copy this map to resume session instantly.
```

---

## Why Layer 2 Is Skipped Here

Light maps skip Layer 2 (Structure) because there's no project structure to document.
The map stays compact — only what's needed to resume.

---

## What Resuming Looks Like

**New session. User pastes the map above.**

Claude responds:

> "Resuming SE to PM Career Switch — covering how to build a PM portfolio while still in an SE role."

Then immediately gives concrete steps for portfolio building — matching the user's preference for actionable, practical advice.

No "tell me about your background." No re-asking about the timeline. No re-debating internal vs external.
**One line. Then execution.**
